# Tropical-cyclone vortex gravity waves

## Description

This repository contains a pseudo-spectral linear shallow-water model for
gravity waves generated by a balanced Rankine tropical-cyclone vortex. The
Python implementation uses Fourier derivatives, an exact Fourier-space
gravity-wave substep, RK4 base-flow coupling, a boundary sponge, and an
optional two-thirds spectral filter. It supports the original NumPy backend,
a PyTorch CPU/GPU backend, and multi-GPU execution with `torchrun`.

The simulation writes total-field snapshots (`u`, `v`, and `h`) as
compressed NumPy files and the outward perturbation power flux through a
configured circular contour as a CSV time series.

## How to run

On the IU GPU system, prepare the requested Python environment first:

```bash
conda deactivate
module load python/gpu/3.12.5
```

PyTorch is included in this module. From the repository root, install any
missing project dependencies:

```bash
python -m pip install -r requirements.txt
```

Edit `src/config.yaml` to set the grid, vortex, integration, diagnostic, and
output parameters.

### NumPy and single-GPU runs

Run the original NumPy experiment:

```bash
python src/tc_gravity_waves.py
```

Run on one GPU:

```bash
python src/tc_gravity_waves.py \
    --backend torch --device cuda --dtype float32
```

`float64` most closely reproduces the NumPy result; `float32` is generally
faster and uses half as much field storage, but its accuracy should be checked
for the intended experiment.

### Multiple GPUs with torchrun

Inside an allocation with four visible GPUs, launch one rank per GPU:

```bash
torchrun --standalone --nproc-per-node=4 \
    src/tc_gravity_waves.py \
    --backend torch --device cuda --dtype float32 --ddp
```

The model has no trainable parameters or gradients, so standard neural-network
`DistributedDataParallel` is not applicable. The `--ddp` option instead uses
`torch.distributed`: each rank owns a complete model state, independent FFT
work is divided among ranks, NCCL collectives synchronize the completed fields,
and only rank 0 writes snapshots and diagnostics.

This replicated-state approach is most useful for larger grids. For the
default 300√ó300 grid, communication can cost more than the distributed FFT work;
benchmark one GPU against the requested number of GPUs before a long run.

The same options can be stored under `compute` in `src/config.yaml`.
Command-line values take precedence. To use another configuration:

```bash
python src/tc_gravity_waves.py --config path/to/config.yaml
```

Relative output paths are interpreted relative to the selected YAML file.
With the supplied configuration, results are written to `outputs/`. After
the simulation finishes, start Jupyter and run the plotting notebook:

```bash
jupyter notebook plots/plot_model_outputs.ipynb
```

In the animation section, set `variable` to `"u"`, `"v"`, or `"h"`.
